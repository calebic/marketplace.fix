Marketplace Fixer Enhanced V1.0 - Update 2

Focus: bulk autofill responsiveness

Changes in this update:
- reduced per-item UI churn during Auto Fill All by removing most intermediate item-state updates
- throttled bulk progress updates so the overlay/status text is not refreshed excessively
- moved grouped batch writes onto a background task instead of blocking the window thread
- made workflow batch writes use a thread-safe snapshot of UI options (backup toggle, vehicles mirror toggle, mods path)
- kept single-item autofill behavior intact while making workflow batch completion updates more compact
- applied the same throttled progress behavior to retrying reviewed mods
- stopped the overlay helper from explicitly disabling the main workspace and title bar while a run is active

Expected result:
- Auto Fill All should still take time on large repos, but it should feel less frozen and less chatty
- window/UI status updates should happen in larger, more meaningful steps
- grouped save/write work should interfere less with general UI responsiveness
