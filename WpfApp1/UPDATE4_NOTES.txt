Marketplace Fixer Enhanced V1.0 - Update 4 (scanner / data-model pass)

This update starts the safer mod-first scanner transition without rewriting the whole UI.

What changed:
- Added ModScannerService with mod-level scan records.
- Scan now builds a mod inventory first, then config items from each mod.
- Added ModContentKind (Unknown / Vehicle / Map / Mixed).
- Added ModScanRecord tracking for each scanned folder/zip mod.
- Scan now detects map-capable mods using source structure / level markers.
- Mod Library and workspace summary can now include map-only mods that do not expose vehicle configs.
- Dashboard/install summary now uses mod-level scan data when available.
- Added SourceCategory to VehicleConfigItem so config items know whether their source mod scanned as Vehicle / Map / Mixed / Unknown.
- Search filter now also matches SourceCategory.

Intent of this update:
- move the app one step closer to real mod-first scanning
- improve visibility of map-only and mixed-content mods
- give the workspace truer per-mod counts instead of only config-derived counts

Important note:
- this update is intentionally a safe first scanner pass, not a full UI/data rewrite.
- core editing still operates on VehicleConfigItem, but the scanner now keeps extra mod-level structure for future updates.
