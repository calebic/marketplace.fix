Update 7: save integrity + canonical lookup correctness

Included in this pass:
- Single-config Save now stages JSON first and only commits live item state after the write succeeds.
- TryBuildUpdatedConfig now works from a deep-cloned JSON object instead of mutating the selected config in place.
- Auto Fill All workflow now prepares staged per-item writes and only commits inferred item state after the batch write succeeds.
- Bulk Edit now uses staged working copies so failed writes do not leave in-memory item state falsely updated.
- Manual Internet Lookup now returns candidate-derived canonical identity fields (brand/model/body style/year when available) instead of always echoing the raw user query text.
- Trusted lookup candidates now preserve page-title/canonical identity metadata so ranking/evidence can reflect the matched vehicle rather than only the search text.

Intended user-visible effect:
- Failed saves should no longer leave the workspace pretending a config committed when disk write failed.
- Manual lookup results should feel more like matched vehicle identities and less like echoed search input.
