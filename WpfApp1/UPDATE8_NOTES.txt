Update 8: Persistent diagnostics + app-state hardening

What changed:
- Added a shared AppPaths helper so settings, crash logs, scrape logs, review state, mod memory, and pricing cache all use the same BeamNGMarketplaceConfigEditor app-data root.
- Added atomic app-state writes with backup fallback via AtomicFileIO for:
  - settings.json
  - mod-memory.json
  - pricing-cache.json
  - mod-review-ignored.json
- Added lightweight state logging to state.log for non-blocking persistence failures.
- Expanded persisted config memory to retain newer diagnostics and history fields, including:
  - ConfidenceScore
  - ConfidenceTier
  - IdentityEvidence
  - ValuationEvidence
  - ReviewReason
  - DecisionOrigin
  - LastDecisionUtc
  - LastLookupSourceName
  - LastLookupSourceUrl
  - LastLookupUtc
- Manual Internet Lookup now records source/time history on the config item.
- Inference/review flows now stamp a durable decision origin + decision timestamp.
- Review ignore/retry actions now stamp decision origin metadata too.
- Diagnostics panel summary/evidence text now surfaces origin/update time plus last lookup trail and source page when available.
- Crash logs were moved onto the same shared app-data root as the rest of the workspace state.

Intent of this update:
- make confidence/evidence/review data survive relaunches
- reduce corrupt/partial app-state writes
- make the workspace feel more durable and explainable across sessions

Caveat:
- This was patched carefully at source level in-chat, but I could not run dotnet build in this environment because dotnet is not installed here.
